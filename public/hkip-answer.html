Q2

We are looking for one WordPress developer to figure out how to do a “Limited viewable CV profile ”function on our website. How do you find this project? It’s that difficult for you?

I do not understand the exact meaning of "Limited viewable CV profile". Therefore I will list some possible features below.

  • Limit specific CV profile contents, only viewable for the specific user role.

    1. Store user role after login into the localStorage (Empty for guest user)
    2. SELECT specific contents depending on user role. (Handle in client app)

  • Limit CV page visibility for specific user

    1. Set up JWT checking functionality for the route that requires authorization.
    2. Send and store JWT in cookies after user login / sign-in
    3. Check user JWT for authorization when user hit the route

  • CV owner change visibility of profile element

    1. Add an "IsVisible" attribute to the element table that allows optional visibility
    2. SELECT only the content WHERE isVisible is true

It would not be too difficult to build only the feature in code (JavaScript from my experience). However, it would take some time to make features such as authorization and restricted routing reusable

Q5

If we hire you as our intern, how do you suggest we improve our website? (3 suggested points, and why ?)

  1. Add an overview profile page

    • Currently the website has the functionality to fill in the profile and resume on different pages, and CV. As the website is not a single page application, it takes time to browse between profile pages, check and modify the content. Therefore, my suggestion is to add an overview page, with docks containing the preview of profile elements respectively. If the user finds mistakes in preview content, use a link to direct user to the specific tab.

  2. Modify the session.

    • Currently the website will store a PHP session cookie after the user logged in, and the cookie will expire when user end browsing, which means every time user enter the site fresh, they will need to log in again. Using this approach maybe because of security concerns, but as a website user, I would prefer auto-login, such that I could check information updates every time entering the site.
    • Therefore my preferred approach is to set the cookie expiration time to 30 or 60 days, update the cookie then user log in using the login route.

  3. Add customization options

    • The website already looks excellent in styling. Website customization could be added to improve user experience.

      • Add font size option

        • Website font is a bit large for me. Therefore, adding font size adjustment to the website would be more convenient than adjusting using the browser setting.

      • Store the user's preferred job category, and provide an additional link for user to fast forward to the filtered result.

Most of the suggested features are to reduce client load by shortcuts.
